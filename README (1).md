## 🔬 Overview

This repository accompanies the paper:

> **"Machine Learning Prediction of Metal Recovery in Bioleaching: A Leak-Free Cross-Validation Study with Uncertainty Quantification"**  
> Hamid Abdoli et al. — *[Journal Name]*, 2025

We trained and evaluated **7 regression models** on a dataset of **840 bioleaching experiments** covering **21 metals**, using a fully leak-free preprocessing pipeline (target encoding and feature selection fitted inside each CV fold). Key contributions:

- ✅ **Leak-free preprocessing**: target encoding & feature selection fitted inside each fold
- ✅ **Honest extrapolation test**: Leave-One-Element-Out (LOEO) protocol
- ✅ **Conformal prediction intervals**: distribution-free 90% coverage
- ✅ **Statistical model comparison**: Friedman test + Wilcoxon/Holm pairwise
- ✅ **Feature-group ablation**: decomposes what drives recovery (process vs element identity)
- ✅ **Live prediction dashboard**: interactive app for new conditions

## 📊 Key Results

### Model comparison (5×5 repeated stratified CV, interpolation)

| Model | MAE mean ± std | 95% CI | R² | CCC |
|---|---|---|---|---|
| **HistGB (tuned)** | **8.34 ± 0.72** | **[7.62, 9.06]** | **0.793** | **0.887** |
| XGBoost (tuned) | 8.77 ± 0.73 | [8.04, 9.50] | 0.767 | 0.871 |
| LightGBM | 9.19 ± 0.48 | [8.71, 9.67] | 0.756 | 0.862 |
| RandomForest | 11.22 ± 0.61 | — | 0.712 | 0.840 |
| Ridge | 15.25 ± 0.92 | — | 0.612 | 0.731 |
| Dummy (baseline) | 26.73 ± 0.81 | — | ~0.000 | ~0.000 |

> **Note on interpolation vs extrapolation:**  
> Hold-out MAE (6.36) appears better than CV (8.34) due to random-split optimism.  
> GroupCV MAE (~26.7, R²=−0.44) reveals that **current models cannot extrapolate to unseen elements** — an honest finding discussed in the paper.

### Feature-group ablation (what drives recovery?)

| Feature set | CV MAE |
|---|---|
| Process only (pH, T, time…) | ~11–12 |
| Element properties only | ~14–15 |
| Element identity only (TE) | ~13–14 |
| Process + element properties | ~9–10 |
| **Full (all + TE)** | **~8.3** |

Process parameters carry the **dominant predictive signal** alongside element identity.

## 🚀 Quick Start

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/bioleaching-ml.git
cd bioleaching-ml
pip install -r requirements.txt
```

### 2. Run the live dashboard

```bash
streamlit run app.py
```

Opens at `http://localhost:8501`. Set your dataset path in the sidebar.  
The model trains automatically on first load (~30 seconds), then is cached.

### 3. Run the full analysis pipeline

```bash
# Edit DATA_PATH at top of file first
python src/bioleaching_ablation_v5.py
```

Outputs saved to `bioleaching_v5_outputs/`. Each block saves incrementally —
a partial run still yields the key tables.

### 4. Run standalone plots

```bash
python src/bioleaching_analysis_plots.py
```

Produces parity plot, residual diagnostics, SHAP, PDP/ICE, per-element error bar.

## 🔑 Methodological Highlights

### Leak-free preprocessing
All preprocessing (target encoding of `element_symbol`, variance thresholding,
ExtraTrees-based feature selection) is fitted **exclusively on training data** within
each cross-validation fold. This avoids a common but critical error in ML pipelines
where preprocessing uses future/test information.

### Conformal prediction intervals
We use **split-conformal prediction** (Angelopoulos & Bates, 2022) with α=0.10
(90% target coverage). The conformal quantile is calibrated on a held-out
calibration set, providing distribution-free marginal coverage guarantees.

### Interpolation vs extrapolation
We distinguish two evaluation regimes:
- **Interpolation** (random CV): test samples share element identities with training
- **Extrapolation** (GroupCV / LOEO): test element is entirely absent from training

The large gap (RandomCV MAE ~8.3 vs GroupCV MAE ~26.7) is explicitly reported
and discussed as a scientific limitation — a common omission in the bioleaching
ML literature.
